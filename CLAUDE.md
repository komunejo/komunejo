# komunejo — development space

This repository is the plugin itself (machinery), not a collaboration space.

## Language convention

All files in this repository are written in English, examples included. Spaces generated by the plugin speak the language their profile declares; the plugin itself never does.

## Working documents

`docs/design.md` is the authoritative design: the kernel’s entity model, the creation flow, skill registration, particularization, and the open questions. Read it before changing anything. Design decisions belong to the user; do not resolve an open question by implementing a default.

## Repository discipline

- Never commit without the user’s explicit confirmation.
- The entity index lives at `entities/entities.md` (folder-note form, this repository’s preference) and is always regenerated with `entity_lint.py index --write entities/entities.md`, never hand-edited. No index at the repo root.
- The bundled `entity-management` skill is vendored (MPL-2.0), pinned to a tagged upstream release. Never fork or edit the engine here — improvements go upstream, and updating the pin is a deliberate, recorded act.
